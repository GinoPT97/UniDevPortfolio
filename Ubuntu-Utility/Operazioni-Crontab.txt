Per eseguire i tuoi script Aggiornamenti.sh e Ottimizzazioni.sh ogni ora senza richiedere la password di sudo, è necessario configurare correttamente il file /etc/sudoers. Ecco come procedere:

✅ 1. Modifica del file sudoers per eseguire comandi senza password
Apri il file sudoers in modalità sicura:

sudo visudo

Aggiungi la seguente riga alla fine del file:

kenobi ALL=(ALL) NOPASSWD: /home/kenobi/Documenti/GitHub/UniDevPortfolio/Ubuntu-Utility/Aggiornamenti.sh, /home/kenobi/Documenti/GitHub/UniDevPortfolio/Ubuntu-Utility/Ottimizzazioni.sh

Questa configurazione permette all'utente kenobi di eseguire i due script senza dover inserire la password.

Salva e chiudi l'editor:

Se stai usando nano, premi Ctrl + X, poi Y per confermare e Invio per salvare.

✅ 2. Modifica della crontab per eseguire gli script ogni ora
Apri la crontab dell'utente kenobi:

sudo crontab -e

Aggiungi le seguenti righe alla fine del file:

0 * * * * /home/kenobi/Documenti/GitHub/UniDevPortfolio/Ubuntu-Utility/Aggiornamenti.sh >> /home/kenobi/Documenti/Aggiornamenti.log 2>&1

0 * * * * /home/kenobi/Documenti/GitHub/UniDevPortfolio/Ubuntu-Utility/Ottimizzazioni.sh >> /home/kenobi/Documenti/Ottimizzazioni.log 2>&1

Queste righe configurano l'esecuzione degli script ogni ora, redirigendo l'output e gli errori nei rispettivi file di log.
