\section{Contesto e Introduzione}
\begin{tcolorbox}[colback=gray!10, colframe=black!40!black, title=Descrizione generale]
Il progetto consiste nello sviluppo di un sistema informativo, composto da una base di dati relazionale e un applicativo Java dotato di GUI (Swing o JavaFX), per la gestione di un negozio di ortofrutta. Il sistema permette di gestire le vendite e la disponibilità dei prodotti, identificando le tipologie (frutta, verdura, farinacei, latticini, uova e confezionati), ognuna con caratteristiche specifiche (ad esempio, la frutta e la verdura hanno una data di raccolta, i latticini hanno sia una data di mungitura del latte che una data di produzione, ecc.).
\end{tcolorbox}

Ogni cliente è registrato e possiede una tessera punti che lo identifica presso il fruttivendolo: per ogni acquisto riceve il 10\% del valore della spesa in punti fedeltà. Il sistema consente la ricerca dei clienti, differenziandoli in base alle categorie di prodotti acquistati e alla quantità di punti ottenuti per ciascuna categoria.

\textbf{Traccia d'esame: Sistema di gestione per un negozio di alimentari}

\begin{tcolorbox}[colback=yellow!10, colframe=orange!80!black, title=Obiettivo della traccia]
Per gruppi da 3: il fruttivendolo ha un certo numero di dipendenti e ad ogni vendita è associato il dipendente che l’ha portata a termine. È quindi possibile ricercare quale dipendente ha effettuato il maggior numero di vendite in un determinato periodo, differenziandolo da quello che ha portato il maggior introito.
\end{tcolorbox}

\section{Architettura del Sistema}
Il progetto adotta il pattern architetturale Model-View-Controller (MVC), separando chiaramente la logica dell'applicazione, l'interfaccia grafica e l'accesso ai dati.

\begin{itemize}
\item \textbf{Model}: gestisce le entità come Prodotto, Cliente, Dipendente, Vendita, ecc.
\item \textbf{View}: realizza la GUI in Swing, con schermate per l’inserimento, la modifica e la visualizzazione dei dati.
\item \textbf{Controller}: gestisce l’interazione tra vista e modello, e collega l'interfaccia grafica alle funzionalità.
\end{itemize}

\section{Struttura del Codice}

Il progetto è suddiviso nei seguenti package, in linea con la struttura reale:
\begin{itemize}
    \item \texttt{model} – Classi delle entità principali (Prodotto, Cliente, Dipendente, Ordine, Tessera, ecc.) e relativa logica di dominio.
    \item \texttt{controller} – Logica applicativa, gestione degli eventi e coordinamento tra GUI e modello.
    \item \texttt{daoimplementation} e \texttt{daointerface} – Interfacce e implementazioni per l'accesso ai dati tramite JDBC.
    \item \texttt{dbconfiguration} – Gestione della connessione al database e configurazione.
    \item \texttt{gui} – Componenti Swing per l'interfaccia grafica (schermate di inserimento, modifica, visualizzazione, ecc.).
\end{itemize}

\subsection*{Esempio di codice (classe Cliente)}

\vspace{0.5em}
\noindent\textbf{Esempio: classe Cliente e calcolo punti fedeltà}

La classe \texttt{Cliente} rappresenta un cliente registrato, dotato di una tessera punti. Il metodo \texttt{calcolaPunti} restituisce i punti fedeltà maturati in base alla spesa effettuata (10\% del valore).

\lstset{language=Java, breaklines=true, basicstyle=\ttfamily\small, frame=single, backgroundcolor=\color{gray!10}}
\begin{lstlisting}
public class Cliente {
    private String nome;
    private String cognome;
    private String tesseraPunti;

    public Cliente(String nome, String cognome, String tesseraPunti) {
        this.nome = nome;
        this.cognome = cognome;
        this.tesseraPunti = tesseraPunti;
    }

    /**
     * Calcola i punti fedelta maturati in base alla spesa.
     * @param spesa importo della spesa effettuata
     * @return punti maturati (10 percento della spesa)
     */
    public int calcolaPunti(double spesa) {
        return (int)(spesa * 0.10);
    }

    // Getter e Setter
}
\end{lstlisting}

\section{Database}
Il sistema si basa su un database relazionale in PostgreSQL. Di seguito si riporta uno schema semplificato delle tabelle principali:

\begin{itemize}
\item \textbf{Prodotto(id, nome, categoria, quantità, prezzo, data\_raccolta, data\_produzione, ...)}
\item \textbf{Cliente(id, nome, cognome, tessera\_punti)}
\item \textbf{Dipendente(id, nome, cognome)}
\item \textbf{Vendita(id, data, cliente\_id, dipendente\_id, totale)}
\item \textbf{DettaglioVendita(vendita\_id, prodotto\_id, quantità)}
\end{itemize}

\section{Conclusioni}
Il progetto ha permesso di mettere in pratica i principi della programmazione orientata agli oggetti, la separazione delle responsabilità attraverso il pattern MVC, e la connessione tra Java e un database relazionale. Ulteriori sviluppi potrebbero includere la migrazione a JavaFX, l'integrazione di report PDF e funzionalità di esportazione dati.
