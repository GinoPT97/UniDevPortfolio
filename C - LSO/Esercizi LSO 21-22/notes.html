Use of ls -F

#!/usr/bin/env bash

echo "part a"
ls -F | grep "^[A-Z].*\/$"
echo "part b"
ls -F | grep ".*a.*a.*\*$"
# alternativa
# ls -F | grep "a.*a.*\*$" # non bloccando l'inizio riga con ^ la prima 'a' puo' trovarsi ovunque nella riga
echo "part c"
ls -F | grep ".*|$" | sed "s/|$//" # oppure sed "/.$//"
echo "part d"
ls -F | egrep -v "(.*\*$|.*@$|.*\|$|.*=$|.*\/$)" | awk '{if (length $0 > 5){print $0}}' # avrei potuto anche fare grep "......*"
# alternativa
# ls -F | grep ".*[a-zA-Z]$" | grep -v ".*=$" | grep "......*"
# alternativa
# ls -F | grep -E ".{6,}.*[^*/@|=]$" #molto piu' elegante
echo "part e"
echo "output: Fi myli"
echo "part f"
echo -e "output: 1 1 1\ncount value at the end: 4"
  • parte e output: Fi myli
  • parte f Supponendo che ls dia “a.out pippo.txt test.txt Files Temp mylib.so myFifo” in questo specifico ordine output: 1 1 1, valore finale di count 4 (non viene stampato). Ordine alfabetico ovvero quello usato di defaul da ls (di solito): “Files a.out myFifo mylib.so pippo.txt test.txt” output 2 3 1, valore finale di count 0

Esercizio 2

Esercizi sui file .fasta. Per il punto b e c ho usato alcune funzioni di awk che non abbiamo visto a lezione, ma che comunque ho usato risolvendo gli esercizi di Ben Porter su awk. Nel dubbio in man awk c’e’ sono elencate molte.

punto a

#!/usr/bin/env bash
echo "part a"

for file in *.fasta; do
    echo $file
    sed "/^>/ s/[^\-\>[:alnum:]]/_/g" $file
    echo -e "\n"
done

punto b

Numerare le sequeze in ogni file 

#!/usr/bin/env bash
echo "part b"
for file in *.fasta; do
    echo $file
    sed "/^>/ s/[^\-\>[:alnum:]]/_/g" $file | awk 'BEGIN{count=1} {if ($0 ~ /^[>-]/) {printf("> %d %s", count, substr($0, 2)); count++} else print $0}'
    echo -e "\n"
done

punto c

Estrarre tutte i blocchi (header e sequeze) 

#!/usr/bin/env bash
echo "part c"
for file in *.fasta; do
    echo $file
    awk '{if ($0 ~ /^[>-]/){header=$0;getline sequence; if(length (sequence) > 199){printf("%s\n%s\n", header, sequence)}}}' $file
    echo -e "\n"
done

Esercizio 3

Spring cleaning script. Purtroppo non ricordo bene la traccia, per quel che ho scritto sugli appunti dovrebbe essere qualcosa del genere:

  • Data una folder ed un’estensione di file, rimuovere tutti i file con quell’estensione (anche nelle sottocartelle).
  • Se la folder/subfolder e’ vuota la si elimina

Controllo parametri

#!/usr/bin/env bash

if [ $# -ne 2 ]; then
    echo -e "Usage: spring_clean.sh start_folder file_extension (without dot i.e. html not .html)\nFor the start folder I need the absolute path"
    exit 1
fi

Funzione di clean

L’algoritmo controlla ogni file nella cartella corrente, se non e’ un link simbolico ed e’ un file regolare che matcha l’estensione data, viene eliminato, se il file in questione e’ una cartella chiama ricorsivamente la funzione nella sottocartella. 

dfs_clean(){
    cd $1
    for elem in *; do
        if [ -f $elem -a ! -L $elem ]; then
            remove=$(echo $elem | awk '{if ($0 ~ /\.'$2'$/) {printf "remove"} else {printf "dont_remove"}}')
            if [ $remove = "remove" ]; then
                rm -v $elem
            fi
        elif [ -d $elem ]; then
            dir=$elem
            dfs_clean $dir $2
            remove_dir=$(ls -A $dir)
            if [ -z $remove_dir ]; then
                rmdir -v $dir
            fi
        fi
    done
    cd ..
}

“Main”

start_dir=$1
extension=$2

pushd $start_dir 1>/dev/null
dfs_clean "." $extension
popd 1>/dev/null

Condiderazioni

Bash usa uno scope dinamico, infatti nelle chiamate ricorsive il cambio di valore su elem e dir si riflette sui chiamanti. Sul manuale di GNU bash non ho trovato un modo per evitare questo problema. Ora dato che che fra caller e callee i valori cambiano mi aspettavo qualche errore, che qualche cartella non venisse ispezionata, invece l’unica cosa che accade e’ la shell stampa degli errori perche’ cerca di cancellare piu’ volte le stesse cartelle. Potevo evitare di stampare gli errori avrei potuto scrivere dfs_clean "." $extension 2>/dev/null, ma nascondere gli errori non credo sia una buona prassi.
